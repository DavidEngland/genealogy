ID EXTRACTION PROCESS

1. Parser reads GEDCOM file
2. Searches for ID patterns in:
   - NOTE fields (where genealogists put references)
   - SOUR (source citations)
   - _UID (external unique IDs)
   - Custom genealogy fields
3. Recognizes patterns:
   - WikiTree: "England-1357" or "England-1357"
   - FamilySearch: "LDLZ-17W" or "LDLZ-17W"
   - Ancestry: "123456789" (6+ digits)
   - Find A Grave: "12345678"
4. Maps GEDCOM internal IDs to external IDs
5. Generates CSV report showing:
   - Which GEDCOM ID mapped to which external ID
   - Person's name
   - Which services provided the ID

WORKFLOW 3: Find Coverage (WikiTree vs FamilySearch)

  schema/process_gedcom.sh --all
  cat data/*-ids.csv | grep -v "^GEDCOM" > all-ids.csv
  
  WikiTree only:     grep ',wikitree' all-ids.csv | wc -l
  FamilySearch only: grep ',familysearch' all-ids.csv | wc -l
  Both:              awk -F',' '$4~wikitree/ && $4~/familysearch/' all-ids.csv | wc -l

WORKFLOW 4: Create Master Index

  schema/process_gedcom.sh --all
  cat data/*-ids.csv | grep -v "^GEDCOM" | sort -u > master-ids.csv
  
  Now have single file with all people and their IDs across all GEDCOMs.
